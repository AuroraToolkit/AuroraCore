Protocols

The following protocols are available globally.

LLMServiceProtocol

  • LLMServiceProtocol

    The LLMServiceProtocol defines the interface for interacting with different LLM (Language Learning Model) services.

    Conforming types (e.g., OpenAIService, AnthropicService, OllamaService) must implement this protocol to enable communication with their respective LLM backends.

    This protocol ensures that all LLM services handle requests and responses consistently, allowing the client to interact with multiple LLMs in a unified way.

    Note

    Each service can define its own token limits and handle authentication in its unique way.

  • LLMResponseProtocol

    A protocol that defines a common interface for the response of different LLM services.

    Conforming types must implement:

    • text: The main textual response generated by the LLM.
    • model: The model used to generate the response, if available.
    • tokenUsage: Optional token usage data, providing information about token consumption during the request.

    This protocol allows for consistent interaction with different LLM services while accommodating their unique response formats.
